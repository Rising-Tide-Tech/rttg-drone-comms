# RTTG Drone Comms Privacy Policy

Last updated: 2026-05-26

This privacy policy describes how the **RTTG Drone Comms** Chrome extension ("the extension") handles data.

## What the extension does

The extension assists Rising Tide Technology Group drone pilots in posting standardized operational status messages to Google Chat spaces. It runs locally inside the user's Chrome browser. It is intended for use by RTTG personnel and authorized contractors only.

## Data the extension processes

The extension reads and stores the following data locally inside Chrome:

- **Pilot profile** entered by the user: name, callsign, phone number, email. Used to auto-fill outgoing message templates.
- **Shift chat list**: display name and URL of the Google Chat spaces the pilot posts into. Entered by the pilot or pushed by an administrator.
- **Templates, missions, and shift time blocks**: pre-configured messages and dropdown values used to construct outgoing messages.
- **Extension settings**: pilot preferences.

All of the above is stored in Chrome's built-in `storage.sync` and `storage.local` areas. It synchronizes across the user's own Chrome installations under the same Google account, governed by Chrome's own sync policy. It is not transmitted to any RTTG-controlled server unless the user explicitly chooses to.

## Network traffic generated by the extension

- **Google Chat (chat.google.com / mail.google.com/chat)**: the extension posts the user-composed message into the user's own Google Chat session, using the user's existing browser authentication. The extension does not obtain or transmit Google credentials. Messages are delivered via Google Chat's own infrastructure, not via any RTTG server.
- **Admin-configured remote config URLs** (optional, not required to function): if an administrator has provided a URL pointing to a hosted JSON configuration file (templates, chat list, DOM selectors, master configuration), the extension fetches that URL on startup and after the user clicks "Sync now." Only the URL configured by the administrator is contacted.
- **Slack incoming webhook** (optional, not required to function): if the user clicks "Report a bug," the extension POSTs the user-entered bug description plus the pilot's profile fields to a webhook URL configured by the administrator. The user controls when this happens by clicking the bug-report button.

The extension does not contact any other endpoints. No analytics, telemetry, advertising, or tracking pixels are present.

## Permissions requested

- `storage` - to persist the pilot profile, templates, chat list, and settings locally.
- `sidePanel`, `tabs`, `activeTab`, `scripting` - to display the side panel and inject the message into the user's open Google Chat tab.
- `clipboardWrite`, `offscreen` - to write to the clipboard as a fallback when DOM injection fails.

Host permissions are limited to chat.google.com, mail.google.com/chat, and hooks.slack.com for the bug-report webhook.

## Data retention

All data remains inside the user's Chrome browser. Removal of the extension removes all local data. The administrator's hosted config files contain no pilot-identifying data.
